1. Node.js의 특징
    - 영화 티켓을 사려는 고객들의 예시
    - 1명이 수백장을 구매하고자 하면 그 뒤 고객은 대기
    - Node.js는 non-blocking IO를 사용함으로써
      모든 요청을 한 번에 받고 빨리 처리되는 것은 빨리 리턴
    - 비동기 IO를 처리 / 이벤트 구조
    - 빠른 속도 : 구글의 V8 엔진
    - 단일 스레드 / 뛰어난 확장성
      이벤트 루프 + 단일 스레드
      이벤트 루프는 서버가 멈추지 않고 반응하도록 해줌

    - 주 사용처
        SNS 또는 챗 서비스에 많이 사용됨(요청은 많고 단위는 가벼움)
        일반 서버가 이 기능을 구현하려면 멀티 스레드를 써야 함
        Node.js는 요청이 많거나 오래 걸리는 요청이 있어도
        대기 시간이 발생하지 않음
        => 개수는 많으나 크기가 작은 데이터를
           실시간으로 주고받는 어플리케이션에 최적
           > 실시간 채팅, 주식 차트

    - 단점
        모든 처리를 단일 스레드가 하기 때문에
        코드가 많은 CPU 연산을 요구할 경우 감당불가
        이미지, 비디오, 대규모 계산 등에는 절대 권장하지 않음

2. Node.js를 실행해보기
(1) 설치
    - Node.js 홈페이지에서 LTS 버전 설치
    - cmd 실행 후 node -v, node --version으로 확인        

(2) js 파일 실행 연습
    - src 폴더 생성
    - main.js를 만들고 console.log("Hello, world"); 작성
    - cmd 실행 후 해당 디렉토리로 이동
    - node main.js 실행

(3) 웹 서버 띄우기
    - require 함수로 http 모듈 불러오기
    - var http = require("http")
      require는 특정 라이브러리를 사용할 때 선언
    - http.createServer 메서드를 이용하여 웹 서버 생성
      .listen(포트번호) 메서드로 포트 번호에서 대기
    - 웹 브라우저에서 localhost:10001 접속 후 
      cmd 및 브라우저에서 확인

3. Node로 구축된 대표적인 사이트
    - 넷플릭스 : 대규모 사용자 트래픽 처리
    - 우버 : 실시간 위치 추적, 주문 처리에 사용
    - 페이팔 : 결제 처리 시스템, 높은 처리 성능

    - 네이버 : 쇼핑, 블로그
    - 배민 : 배달 서비스의 실시간 주문 처리
    - 나무위키 : Node.js로 검색 기능

4. NPM(Node Package Manager)
    - Node.js의 패키지 설치 및 버전/호환성 관리
    - npm install <설치할 모듈명>   cf) in python : pip install 모듈명

5. Callback
    - 특정 작업을 하도록 함수를 전달하는 것

    건너마을 아저씨가 빵을 주신단다.
    우리 아들을 거기에 빵을 가져오게 보낼 예정
    빵 받고 오면 딩동 해라 -> 빵 받거든 할머니에게 전달까지 하고 와

    내가 가서 가져옴
    가져와서 할머니에게 전달 -> 가져와서 전달할 때까지 아무 일도 못함

6. Event Loop
    - Event Emitter(이벤트 발생)
    - Event Queue에 쌓임(Queue의 특징 : FIFO)
    - Event Loop(Observer) : Event Catch
    - Event Handler

    이벤트 발생 > 이벤트 감지 > 이벤트 실행
    이벤트 방식의 node.js 구조는 빠르다 는 장점을 가짐
