Day1

1. Web
(1) Web의 기본 목적
    다른 컴퓨터에서 문서를 공유하거나 보는 목적
(2) Web의 구조
    인터넷을 활용하여 거미줄처럼 연결, WWW(World Wide Web)
    웹 문서를 인터넷 상의 컴퓨터끼리 주고받는 네트워크 시스템
(3) Web의 구성
    웹 서버 - 웹 클라이언트

    1) 클라이언트
        사용자 인터페이스 담당
        웹 서버에 웹 문서를 요청

    2) 웹 서버
        웹 사이트(문서)를 탑재한 컴퓨터
        웹 문서, 이미지, 동영상 등 데이터를 저장

    3) 웹 컨테이너
        동적인 데이터 연산 요청이 들어왔을 때 
        서버가 연산을 요청하는 곳
        연산이 끝나면 정제된 데이터로 서버에 돌려줌

    4) WAS(Web Application Server)
       서버 + 컨테이너
       가장 유명한 것은 Apache Tomcat

2. 인터넷(Internet)
    웹의 개념이 나오기 전 만들어진 컴퓨터 연결 네트워크
    1969년 미 국방성 고등 연구 계획국(ARPA)
    여러 대학교와 계약 업체 사이의 컴퓨터 연결
    컴퓨터마다 고유한 주소(IP Address)를 부여하여 컴퓨터를 구분

3. WWW
    인터넷을 활용하는 응용 서비스 중 하나
    웹 서버와 웹 클라이언트(브라우저)로 구성되는 
    정보 전달 및 공유 서비스
    인터넷은 고속도로, 웹은 물류배송 서비스

4. 웹 브라우저
    Netscape Navigator
    - 일반인들도 쉽게 사용 가능한 GUI를 갖는 최초의 브라우저

    Internet Explorer
    - 1995년 마이크로소프트가 개발
    - 윈도우 운영체제에 끼워서 배포, 순식간에 Netscape 잠식

    Safari
    - 2003년 Apple에서 개발
    - Mac OS, 모바일 ios에서 실행

    FireFox
    - 2002년 개발, 웹 표준기구(W3C)의 표준에 가장 충실
    - 개발자 도구 대폭 강화
    - Netscape가 개발하고 소스 코드 공개

    Chrome
    - 2008년 구글에서 개발, 새로운 강자, 현재 가장 많이 사용

    Microsoft Edge
    - Microsoft가 IE를 포기하고
      Chromium을 인수하여 확장 개발한 브라우저

5. 웹의 성공
    - Tim Berner-Lee의 아이디어
    - 1989년 웹 개념을 제안 > 1990년 WWW 프로젝트 시작
    - 서버와 클라이언트로 동작하는 http 모델 창시
    - 웹 페이지는 텍스트 문서이며 편집기는 아무 것이나 사용 가능
    - http
        웹 브라우저가 웹 페이지를 다 전송받고 나면 웹 서버와 접속이 끊어짐
        웹 서버와 웹 브라우저의 역할 분담
        웹 서버의 낮은 부하로 많은 동접자를 지원 가능
